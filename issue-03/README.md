# 🚨 트러블슈팅 미션 3: 배포가 거부되었습니다!

## 📝 상황 설명
신규 프로젝트를 위해 애플리케이션(`sample-app`)을 배포하려고 `helm install` 명령을 실행했습니다.
하지만 파드가 생성되기는커녕, 터미널에 붉은색 에러 메시지가 출력되며 배포 자체가 거부되고 있습니다.

## 🎯 목표
1. 터미널의 에러 메시지를 꼼꼼히 읽고 배포가 실패한 근본적인 원인을 찾으세요.
2. 클러스터 환경을 조작하거나 코드를 수정하여 애플리케이션이 정상적으로 배포되게 만드세요.

## 🚀 실행 방법
1. **이전 미션 정리:** `helm uninstall mission-2` (아직 지우지 않았다면)
2. **장애 배포:** `helm install mission-3 .`
   *(이 명령어를 치는 순간 터미널에 에러가 발생할 것입니다.)*
3. **상태 확인:** 에러 메시지를 확인하고, 현재 클러스터의 상태를 점검해 보세요.
4. **해결 후 재배포:** 원인을 해결했다면 다시 `helm install mission-3 .` 실행
